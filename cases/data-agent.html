关键判断

NL2SQL 的核心不是把自然语言直接翻译成 SQL，而是先完成业务场景、指标口径、表结构和上下文之间的翻译。没有语义层，Agent 很容易答得像、但不可靠。

难点

基础数据很多，但缺少体系化组织：点位、字段、表、口径、历史需求分散存在，需要先把知识结构化，再谈自动化分析。

沉淀结果

沉淀指标口径、表结构、点位和历史需求的知识化流程，支撑临时取数、口径查询、异动分析和报告草稿生成，提升单人分析效率。

Agent 能做

  • 查询指标口径、点位和表结构
  • 生成 SQL 草稿和分析计划
  • 汇总历史需求和相似案例
  • 辅助异动分析和报告草稿

Agent 不能做

  • 不能虚构不存在的表、点位或口径
  • 不能替代业务判断和结果复核
  • 不能直接决定上线或治理动作
  • 不能绕过权限和数据安全边界

知识库内容

  • 业务场景、界面、链路和常见问题
  • 指标口径、数据词典和枚举值
  • 表结构、字段解释、样例和血缘
  • 历史临时需求、看板和案例库

产出价值

  • 把分析师经验转成可复用流程
  • 设计反问和自校验规则
  • 定义人机协同边界
  • 推动知识补缺和持续更新
