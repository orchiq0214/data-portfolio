关键词驱动的热点发现

Trend Radar：把一个关键词变成带证据的热点清单

围绕一个关键词自动扩展别名、跨源扫描、聚类去重、按证据强度排序，输出可追溯的热点列表。

工作流架构

流程：关键词 → 别名扩展 → 多源扫描 → 聚类去重 → 评分排序 → 热点清单

工作流

关键词输入
别名扩展
多源扫描
聚类评分
热点清单

关键判断

热点发现的核心不是"找到更多"，而是"找到带证据的不同"。没有跨源覆盖和去重聚类，关键词只是噪声放大器。

难点

单源扫描容易遗漏盲区，过度聚合又会把不同主题混在一起。需要的是"广度优先 + 严格去重"的扫描纪律，以及对每个热点附证据和置信度。

沉淀结果

形成标准化的扫描源清单（搜索 / 官方 / 社区 / 长文 / 垂类），并对每个热点附证据链接和置信度标签，避免凭印象判断趋势。

Agent 能做

  • 跨源扫描搜索 / 新闻 / 官方 / 社区 / 长文 / 垂类
  • 聚类近重复主题，去除噪声
  • 按 相关性 + 时效 + 证据强度 评分
  • 输出带链接和置信度的清单

Agent 不能做

  • 不能伪造不存在的趋势或来源
  • 不能跨域聚合不同主题（品牌名 / 通用词混淆）
  • 不能替代人工判断"哪个值得跟进"
  • 不能绕过付费墙或权限边界

源覆盖

  • 搜索 / 新闻：必扫（广度优先）
  • 官方 / 公告：垂类必扫
  • 社区 / 社交：验证热度
  • 长文 / 分析：深度补充
  • 垂类（财报 / 专利 / 职位 / 电商）：按需

产出价值

  • 把"听说很火"变成"可验证清单"
  • 把扫描纪律沉淀成可复用流程
  • 支持中文 / 英文双语境查询
  • 支持品牌名 / 型号 / 事件多形态扩展
