# 示例：某跨境电商公司数据分析经理岗位

*虚拟案例，用于展示 JD深度解读 Agent 的输出格式。*

## 一、公司/行业画像

### 基本信息

| 项目 | 内容 |
|------|------|
| **行业** | 跨境电商（B2C服装/户外用品零售） |
| **商业模式** | 精品品牌型跨境电商——自有品牌设计、生产管控、多渠道分销（亚马逊/独立站/速卖通），赚取品牌溢价与供应链差价 |
| **发展阶段** | 成长期向规模扩张期过渡 |
| **品牌矩阵** | 主品牌（战术户外风格）+ 子品牌（大众户外） |
| **主要渠道** | 亚马逊、独立站、速卖通 |

### 招聘结构分析

假设该公司同时在招以下岗位：

| 职能线 | 在招岗位举例 | 说明 |
|--------|-------------|------|
| 设计/产品 | 设计师、版师 | 有自营设计能力 |
| 供应链/生产 | 生产经理、QC、计划主管 | 供应链管控较深 |
| 数据/分析 | 数据分析经理、初级数据分析师 | 数据体系正在搭建 |
| 市场/渠道 | 媒介投放、渠道经理 | 加大流量端投入 |

**从招聘结构看：** 有自营设计能力和供应链管控能力（非纯贸易型），数据体系正在从0到1搭建。

## 二、业务拆解

### 核心业务链路

```text
市场研究 → 产品设计 → 采购生产 → 仓储库存
→ 多平台铺货 → 流量获取 → 转化成交
→ 跨境履约 → 售后复购 → 利润核算
```

### 关键环节与指标

| 环节 | 目标 | 核心指标 |
|------|------|---------|
| 选品/设计 | 找到高毛利、有差异化空间的需求 | 毛利率、品类趋势 |
| 流量获取 | 以可控成本获得目标流量 | ROAS、ACOS、TACOS、CPC |
| 转化成交 | 最大化每一分流量效率 | 转化率(CVR)、客单价(AOV) |
| 供应链/库存 | 不断货、不滞销 | 周转天数、动销率 |
| 利润核算 | 精确归因到SKU/渠道 | 单品净利率、综合ROI |

### 赚钱最关键的环节

1. **选品与设计**——决定利润天花板，差异化定位支撑溢价
2. **广告投放效率**——跨境电商核心竞争壁垒，ACOS控制直接决定净利
3. **供应链成本**——退货+仓储+平台佣金，管控不好就吃掉所有利润
4. **库存周转**——季节性品类，积压直接吃掉利润

## 三、JD深层解读

### 要解决的核心问题

根据JD描述的职责（数据监控、BI体系建设、广告分析、AI落地），推断当前最痛的三个问题：

**痛点1：数据分散，没有统一经营视图**
- 多平台数据各自管理，口径不统一
- 推测现状：各平台数据导出Excel人工汇总，不同人对"利润"定义不同
- JD第一条就写"搭建指标体系、统一数据口径"的原因

**痛点2：报表靠人工，响应慢**
- 当前还在人工做表或Excel半自动化
- 日/周/月经营分析机制未完全建立
- 管理层看数据滞后，无法及时发现异常

**痛点3：广告归因粗放**
- 多流量渠道（站内广告、站外投放、自然流量），没有系统的归因机制
- 可能停在"看整体ROI"层面，无法精细化到单品

### 面试准备重点

| 优先级 | 方向 |
|--------|------|
| ★★★★★ | 跨平台利润归因——如何把各渠道的利润算清楚 |
| ★★★★★ | BI体系从0搭——指标体系、统一口径、推动跨部门使用 |
| ★★★★☆ | 广告分析——ACOS/ROAS/TACOS的实操理解 |
| ★★★★☆ | AI落地——自动报告/智能预警的具体场景 |

## 四、面试可用话术

### 场景：跨平台利润归因

> "多平台电商最核心的问题是把各平台的利润算清楚。我的思路分三层：
>
> 第一层，统一口径。拉一个全渠道收入成本结构表——各平台的收入、退款、平台佣金、广告费、物流费、采购成本逐项对齐到统一的利润公式。
>
> 第二层，归因模型。和广告团队确定归因规则（末次点击/线性归因/时间衰减），确保每个渠道的广告效果有统一的衡量标准。
>
> 第三层，可视化闭环。利润看板能下钻到单品级，让业务方基于数据做定价、广告预算和补货决策。"

### 场景：BI体系建设

> "分阶段推进：前两个月先出一个MVP看板——把最核心的指标快速可视化。同时做一件最基础的事：拉通指标口径，和运营、财务、供应链一起把每个指标的定义写清楚。口径不统一，看板做得再好也没用。
>
> 体系稳定后，再开始做专项分析和AI赋能。"

### 场景：AI落地

> "AI在电商数据中最务实的落地场景有三个：自动报告与智能预警、知识管理与经验沉淀、广告辅助决策。
>
> 但AI落地的前提是数据基础扎实——口径统一、质量可靠、流程规范。先把基础设施建好，再引入AI，而不是反过来。"
